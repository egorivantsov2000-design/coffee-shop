# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Веб-сайт кофейни "CoffeeBreak"

---

## 1. ХАРАКТЕРИСТИКА ПРЕДМЕТНОЙ ОБЛАСТИ

**CoffeeBreak** — это современная кофейня, специализирующаяся на:
- Премиальных сортах кофе из различных регионов мира
- Свежих десертах и выпечке
- Создании уютной атмосферы для работы и встреч

**Целевая аудитория:**
- Люди 20-45 лет
- Профессионалы и фрилансеры, ищущие место для работы
- Кофейные энтузиасты
- Любители качественного кофе и десертов

---

## 2. ТЕМА РАЗРАБОТКИ

Разработка многостраничного веб-сайта кофейни с каталогом продукции и возможностью ознакомления с характеристиками отдельных сортов кофе.

---

## 3. НАЗНАЧЕНИЕ РАЗРАБАТЫВАЕМОГО ПРОДУКТА

Веб-сайт служит:
- **Витриной** компании в интернете

- **Маркетинговым инструментом** для привлечения новых клиентов
- **Платформой** для взаимодействия с потенциальными партнёрами

## 5. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 5.1 Требования к языкам программирования
- **HTML5** — семантическая разметка, валидный код
- **CSS3** — modern CSS (Flexbox, Grid), без препроцессоров
- **Vanilla JavaScript** (ES6+) — без фреймворков и библиотек

### 5.2 Требования к дизайну

#### Цветовая гамма:
```
Основной коричневый:   #2E241E (очень тёмный коричневый)
Светлый фон:           #F8F5F0 (бежевый)
Акцент золотой:        #D9B382 (теплый золотистый)
Тексты вторичные:      #6B5A4E (тёмный серо-коричневый)
Белый фон:             #FFFFFF
Светлый фон элемента:  #F3EFE9
```

#### Шрифты:
- **Основной:** `system-ui, 'Segoe UI', Roboto, sans-serif`
- **Стиль:** Современный, минималистичный, профессиональный

#### Отступы и радиусы:
- Основной border-radius: `24-32px`
- Карточки товаров: `24px`
- Основной контейнер: `32px`
- Стандартные отступы: `2rem`, `1rem`, `0.5rem`

#### Типография:
- **H1:** 2.2rem, font-weight: 600, letter-spacing: -0.5px
- **H2:** 1.8rem, font-weight: 600
- **Body:** 1rem, font-weight: 400, line-height: 1.6
- **Подписи:** 0.875rem, color: #6B5A4E

### 5.3 Требования к стилю написания кода

#### HTML:
- Использовать семантические теги (`<header>`, `<main>`, `<section>`, `<article>`)
- Структурировать с понятными классами: `.cafe-container`, `.product-card`, `.gallery-main`
- Один класс = одна ответственность
- Использовать data-атрибуты для JavaScript: `data-category`, `data-id`

#### CSS:
- BEM-подобная нотация: `.product-card`, `.product-card__title`, `.product-card--active`
- Группировка правил по логике (layout, typography, colors, effects)
- Mobile-first подход (`@media (min-width: 768px)`)
- Логичный порядок CSS-свойств: position → display → размеры → цвета → эффекты
- Комментарии для сложных блоков

#### JavaScript:
- Camel-case для переменных и функций: `filterProducts()`, `productData`
- Const/let вместо var
- Стрелочные функции где уместно
- Функции 20-50 строк максимум
- Комментарии для нетривиальной логики
- Обработка ошибок с try-catch где необходимо

## 7. СПЕЦИФИКАЦИЯ РАЗРАБОТАННОГО ПРОДУКТА

### Уже реализовано в проекте:

#### CSS классы (index.html):
- `.cafe-container` — основной контейнер
- `.toolbar` — панель фильтров и поиска
- `.filter-group` — группа фильтров
- `.filter-btn` — кнопка фильтра
- `.filter-btn.active` — активный фильтр
- `.search-box` — поле поиска
- `.coffee-grid` — сетка товаров (CSS Grid/Flexbox)
- `.product-card` — карточка товара
- `.product-card__image` — изображение
- `.product-card__title` — название
- `.product-card__info` — информация о товаре
- `.product-card__price` — цена

#### CSS классы (product.html):
- `.product-container` — контейнер страницы товара
- `.back-link` — ссылка назад
- `.gallery` — контейнер галереи
- `.gallery-main` — главное изображение
- `.gallery-thumbs` — миниатюры
- `.product-details` — основная информация
- `.product-specs` — характеристики товара
- `.spec-row` — строка характеристики

#### Структура данных товаров (JavaScript):
```javascript
{
  id: 1,
  name: "Эфиопия Сидамо",
  category: "light",
  country: "Эфиопия",
  roastLevel: "light",
  flavorNotes: ["цветочные", "ягодные", "фруктовые"],
  description: "...",
  price: "450 ₽",
  rating: 4.8,
  image: "images/sidamo.jpg"
}
```

#### JavaScript функции:
- `initFilters()` — инициализация фильтров
- `filterProducts(category)` — фильтрация по категории
- `searchProducts(query)` — поиск по названию
- `renderProducts(data)` — рендер карточек
- `getProductById(id)` — получение товара по ID

## 9. ПОРЯДОК ПРИЕМКИ РАБОТЫ

### 9.1 Процесс Pull Request

1. **Разработчик создаёт Pull Request** с описанием изменений
2. **Администратор проводит code review:**
   - Проверяет соответствие ТЗ
   - Проверяет качество кода
   - Тестирует функциональность
3. **При замечаниях:** разработчик исправляет, администратор ревьюит снова
4. **При одобрении:** администратор мержит PR в main

### 9.2 Чек-лист для администратора

- [ ] HTML валиден и семантичен
